Projects and Runs

In IFTDSS, a project is like a workspace. Within a project, you select a data set and establish runs. Projects help you organize your runs and can be themselves organized by workflow.

Runs

A run defines the activity that you would like to perform and the modules or processes that you will use as part of a run.

Using a directory structure metaphor, you can view a project as your root directory, with runs as sub-directories.  You can have a single run or multiple runs associated with a project.

Project Activities

At a high level, the activities you will be performing are to

  1. Create a new project.
  2. Within a new project, create a run by first choosing the type of run you wish to make.
    • Using the tools available for prescribed burn planning, you can model fire behavior and fire effects and perform other types of analyses (i.e., fire containment and probability of ignition) that support the development of burn plan documentation.
    • Using the tools available for performing a hazard analysis, you can model fire behavior across a landscape and identify areas that may be potentially hazardous if a fire were to occur.
    • Using the tools available for risk assessment, you can examine values at risk, develop values-at-risk maps, and model fire behavior and the expected loss or benefit from fires.
  3. Select the module you wish to run.
  4. Name your run.
  5. Configure the module.
  6. Provide the input data required by that module.
  7. View your output data.
  8. View your run summary.

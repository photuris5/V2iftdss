Accessibility

For accessibility, I have been adding alt text to all figures (when adding the figures to a help page). I started this process in mid-summer 2012, so older images do not all have alt text. When creating alt text, you don't need to repeat what is in the regular text, because a screen reader will read the regular text. Instead, if it would be helpful, describe the unique part of the contents of the figure. At a minimum, describe the figure in a sentence.

When editing regular text, even if something is in a figure, keep in mind that someone might not be able to see that figure, so make sure the regular text repeats or echoes the information in the figure some way. For example, if the figure has a link circled in it, also put the full text of the link into the regular text.

Create separate PDF

I create a PDF version of the help I want by creating a separate target with its own TOC that includes just the topics I need. I would then add the generated PDF document to a Resources subfolder called "PrintableDocuments" and provide links to that PDF from the web topics via links like "Printable Version". This does not generate a PDF on the fly, but that works for me.

Create single topic with "tabs"

Another option is to keep the web help information all in one topic, but display it under different tabs. Users click through different tabs which is a lot easier that scrolling a long topic, however, the content is all stored in the one place - have a look at the site (http://www.barelyfitz.com/projects/tabber/). This site shows examples and provides the CSS and javascript you need to get this to work. It is mainly the CSS that manages the layout - I think you can then print a whole topic that uses the print medium that has each tabbed section printed one after the other.

Set the widths of table columns to percentages.

Use the following percentages for the input/output parameter tables:

<col style="width: 15%;" class="Column-Column1" />

<col style="width: 20%;" class="Column-Column1" />

<col style="width: 65%;" class="Column-Column1" />

Importing and Formatting Word Tables

Mary Jo, you may need to bring tables into Flare from Word. The process is pretty straightforward if you are importing a plain Word table. Therefore, when possible, remove all table formatting—colors, styles, etc. before importing.

Unfortunately, if the table has been formatted in Word, Flare brings in all kinds of messy Word formatting. That formatting needs to be removed at the code level.

To import and format a nice, easy, plain table,

  1. Access Project Organizer > Imports.
  2. Double-click on WordImport. This opens an existing Word import panel. (Alternatively, you can right-click on Imports and select Add MS Word Import File. The Add File dialog box opens; enter a name for the import file and choose Add.)

    I'm assuming you will be using the WordImport file.
  3. Select any files showing in the WordImport panel and choose Remove (bottom right).
  4. Now choose Add Files. Browse to and select the Word file(s) you want to import.
  5. In the upper corner of the WordImport panel, choose Reimport.
  6. Various dialog boxes ask you various things. You can pretty much just accept whatever is offered.
  7. The imported file is placed in the Content Explorer under WordImport.
  8. Click on the file name and press F2 to rename the file according to whatever convention fits where it is going. When asked what you want to do with the file, save it under its new name and remove the link.
  9. Open the file, right-click anywhere inside the table, and choose Table Properties.
  10. You pretty much just need to make two changes in the Table Properties>General tab: set Number of header rows (usually just one), and set Table Styles to Alternating Rows. (Or to one of your three other choices. There are two alternating row styles, one with text left-aligned and one with text centered. There are two plain styles—no alternating row colors—again, one is left-aligned and the other is centered.)
  11. Flare creates a new header row; it isn't smart enough to convert your first row into a header row. So either drag and drop the table header text into the correct cells in the header row, or copy and paste the text, or retype it. Then delete the old header row.
  12. Save everything.
  13. Submit the file change to source control (Source Control tab > Check In All), then move the table file to the folder where it needs to go. (You can just drag and drop.) The reason for saving, submitting, then moving is because Flare gets confused when you try to submit more than one operation on a file at a time to source control. It usually won't check the file in and you have to perform all kinds of shenanigans to get that file into source control. So, for example, if you rename a file and also try to delete it, Flare generally won't cooperate.

To import and format a Word-formatted table,

  1. First, if possible, take the easy route by removing the formatting in Word. If for some reason you can't, do everything you would do for the easy table up through saving everything.
  2. Now, open the file in the text editor. You're going to be doing a lot of searching and replacing.
  3. Take a look at the page title (after the <body> element); make sure it has <h1> before the text and </h1> after, and no odd <p something is here> and/or <span> around or inside the title.
  4. Search for all <p> elements and then again for all </p> elements and replace with nothing.
  5. Search for all <p something is here> elements and replace with nothing. Often there will be a bunch of paragraphs with class="p_1", class="p_2" and so on. You'll need to search for each of these individually.
  6. Do the same for </span> and <span class="something"> etc.; search for them and replace with nothing.
  7. Search for just the word span and take a look; often the global searches don't find them if they are next to other text. Remove every span or p element.
  8. You might have to find <sup something is here> elements and change them just to <sup>.
  9. If there are any paragraphs outside the table, for each one, put <p> in front and </p> at the end.

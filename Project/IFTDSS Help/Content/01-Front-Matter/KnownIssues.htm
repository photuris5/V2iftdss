Known Issues

IFTDSS is under ongoing research and development and should be considered an experimental system. As such, it will be continually updated and changed. The IFTDSS development team is working on resolving these issues, which will be corrected in future releases.

Issues

  • IFTDSS supports landscape data sets on the order of 250,000 acres. We are working to increase the size of landscape (LANDFIRE) data that can be used for landscape-scale analyses.
  • We are working on streamlining the GIS polygon drawing tools and the ability to import locally-developed Shapefiles.
  • IFTDSS is being developed for compatibility with most standard and commonly used web browsers (Internet Explorer, Mozilla Firefox, Google Chrome); however, using older versions of these browsers may cause the system to run more slowly than normal and/or may cause features on the screen to be misaligned or not appear.

Set Flame Length Classes

At the Set Flame Length Classes step for the following pathways, you can only specify flame lengths in feet, even if you switch to metric units.

  • Burn Probability by Random Ignition
  • Risk Assessment by Flame Length Classes
  • Risk Assessment Worst Case Flame Lengths

However, IFTDSS will correctly bin the flame length classes in the units you have chosen.

Probability of Burning

At the Output step for the following pathways, the area outside your data set will be produced as zeros rather than as having no data. This may make it seem as though an area has no probability of burning, when in fact the model was not run on the area at all and so the probability of burning is unknown.

  • Fire Behavior Across a Landscape
  • Burn Probability by Random Ignition
  • Risk Assessment by Flame Length Classes
  • Risk Assessment Worst Case Flame Lengths

Detailed Description of the Issue

On the Configure step for these pathways, if you select a data set that is smaller than your run area, the output values for each pixel in the area within the run set but outside your data set will be set as zeros (rather than no data).

The following images illustrate this issue. In the example used for these images, we chose a data set smaller than our run area. In the first example, the area within the run but outside the data set area is displayed with an overlay that seems to indicate there is zero probability of burning for the area represented by those pixels.  Instead, because there is no data, there should be no overlay at all.

In the second image, the spatial landscape data correctly shows data in the data set area and no data outside of the data set area.

Number of Fire Ignitions to Simulate

For the following pathways, the IFT-FlamMap-RANDIG Input parameter Number of Fire Ignitions to Simulate is limited to 20,000 ignitions:

  • Burn Probability by Random Ignition
  • Risk Assessment by Flame Length Classes
  • Risk Assessment Worst Case Flame Lengths

Using a large data set (approximately 250,000 acres), the IFT-FlamMap-RANDIG module is estimated to take an hour to run per 2,000 ignitions. This is just an estimate; the module could take longer to run based on user load.

Works in Progress

The Fire and Fuels Application (FFA) tools (FCCS, Consume, and FEPS) are under active development and have yet not undergone rigorous testing. User documentation for these tools is also under development. The IFTDSS development team is working with the FFA team to make updates and adjustments to these tools and pathways. These tools will be available for active use in future releases of IFTDSS.

Other BehavePlus 5.0.5 Functionality Not Implemented in IFTDSS Version 1.0

  1. Acceptable fire conditions. With the acceptable fire conditions functionality, users can create the output and build display tables of the ranges of acceptable fire behavior. With this feature, users can select a range of acceptable conditions and build data tables for when the fire will be in prescription.
  2. Display output distances in map units. This functionality runs the slope versus Horizontal Map distance tool and automatically converts the surface fire behavior output, such as spotting distance, to values that can be directly entered on maps.
  3. Viewing fuel model parameters. In BehavePlus, users can view the actual parameters for each fire behavior fuel model. IFTDSS provides a summary table of the fire behavior fuel model parameters for each of the standard fuel models in use (these tables are in this help documentation), but does not provide the ability to directly view or edit fuel model parameters within the system.

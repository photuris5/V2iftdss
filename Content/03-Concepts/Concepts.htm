Concepts

In this section, we provide an overview of workflows, describe how to create projects and runs, discuss data sets and data management, and provide an overview of the scientific modules included in IFTDSS. Core Organization

IFTDSS is organized around projects, data, runs, and modules.

The modules are organized around workflows that address the business needs for fuels treatment planning and management. The full suite of workflows includes prescribed burn planning, hazard analysis, spatially explicit fuels treatment assignment, fuels treatment effectiveness, fuels treatment effectiveness over time, and risk assessment. IFTDSS provides tools to support prescribed burn planning, hazard analysis, and risk assessment.

Projects and Runs

Within these workflows, a project is like a workspace.

  • When you set up a project, you can select a data set to associate with your project.
  • If you are performing a spatial analysis, you define an area of interest for your project.
  • A project can have one or more runs associated with it. A run defines the activity that you would like to perform and the module(s) or processes that you will use as part of a run.

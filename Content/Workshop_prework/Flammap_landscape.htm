Running IFT-Flammap for a Landscape

In this tutorial we'll run fire behavior spatially across the landscape using IFT-Flammap. We will use the LANDFIRE data set unedited, assuming a pre-treatment landscape, and we'll run the model under typical fall environmental conditions. We'll demonstrate IFT-Flammap outputs, and the ability to classify those outputs by percentiles.

As we run IFT-FlamMap we'll need to :

  • Specify a LANDFIRE LCP dataset
  • Select a crown fire calculation method
  • Specify fuel moistures for 1,10, 100 hour fuels as well as live herbaceous and woody fuels
  • Specify wind direction and 20 foot windspeed

Outputs will include:

  • Flame length
  • Rate of spread
  • Fireline intensity
  • Heat per unit area
  • Crown fire activity
  • Mid-flame wind speed
  • Horizontal movement rate
  • Direction of maximum spread

Initiating the Run

From the IFTDSS Dashboard, go to the project summary page for the project we created in Tutorial 1, via the Projects tab, or the Manage My Projects link.

From the Project Summary page, scroll down and select Create New Run. From the list of tasks, select Prescribed Burn Planning>Fire Behavior> and Calculate fire behavior across a landscape (IFT-FlamMap).

Set up

On the Set up page, you will name the run, and have the opportunity to change the extent. By default, runs will use the same extent as the project area of interest. However; if you remember from the introductory videos, you may make runs that are smaller than the project area of interest (but not larger). For this tutorial, we will leave the extent at its default setting.

Give the run a descriptive name, such as 'Squaw Crk. Rx Fire Landscape' and click Next.

Configuration

On the configure page, ensure the Squaw Creek dataset is selected and click Next. This data set is the one we acquired when we downloaded the LANDFIRE data for our project. It is stored in the Saved Data Sets section of IFTDSS.

   Note   

Remember the Saved Data section of IFTDSS (available from the Data tab along the top of IFTDSS) gave us the option to upload shapefiles to IFTDSS. If we had polygons in the Saved Data section that we wanted to upload to this run, that could be done using the Import Polygons option on this page.

Inputs

For this example use Finney as the crown fire calculation method. Specify fuel moisture and weather inputs that are representative of a fall prescribed burn scenario. In this case:

  • 1 hour fuel moisture: 8%
  • 10 hour fuel moisture: 10%
  • 100 hour fuel moisture: 16%
  • Live herbaceous fuel moisture: 45%
  • Live woody fuel moisture: 100%
  • Wind direction at 200 degrees
  • 20-foot windspeed of 8 mph

When completed, the inputs page should look similar to image below. Scroll to the bottom and click Next.

Reviewing Landscape Data

After supplying inputs to the run you may review the landscape data, and make any changes you deem necessary. For this example we will leave the data as-is. However, if we wanted, we could use the mapping toolbar to edit points, or make advanced edits across the landscape at this stage in the run.

Remember from the mapping videos that the point edit tool and landscape statistics tool can be used to further review your landscape data.

   Note   

Edits made to the landscape data within a run will only be saved within that run dataset. They will not be applied to the original Project Area of Interest dataset. However, when edits are made to data in IFTDSS Data Studio, and saved, they will be permanent.

To run IFT-FlamMap, scroll to the bottom and click Next.

IFT-FlamMap outputs

Once the modeling is complete, the outputs page will display fire behavior outputs across the landscape. Take a couple minutes to review the landscape, and change the layers that are visible. We can toggle between the different fire behavior output layers from the Layers menu to the left of the map. The map legend is displayed on the right.

Take a few moments to assess different fire behavior outputs across the landscape. Are the fire behavior outputs what you would expect to see given these vegetation types under the environmental parameters? For this tutorial, we will use these outputs. In your own future runs, you may find the outputs do not match what would be expected, and returning to the Review Landscape Data step, making adjustments, and re-running, may be warranted.

In the next steps, these outputs will be classified by percentile and displayed. To do this, scroll down and click Next.

Classifying IFT-FlamMap Outputs

This step allows you to classify fire behavior outputs into more meaningful bins for further analysis and display. Specify the minimum value for each class of each parameter. You can do this by electing to accept default minimum values or by modifying minimum values. IFTDSS automatically bins outputs into three pre-identified classes (Medium, High, Very High). The minimum value for the Low class of each parameter is always zero and thus is not editable. For this tutorial we'll use the default values.

In addition to classifying fire behavior outputs, you may also group outputs into relative percentages for each parameter. By assigning relative percentages you can narrow your search to areas with the greatest potential fire hazard. IFTDSS default values for all categories is 20%. For this tutorial we have left these as their default values. For your own future projects, note that modifying the High and Highest categories to low percentages (5-15%) will help focus attention on the highest fire hazard areas. The sum of all five categories for each fire behavior parameter must equal 100%.

To view the classification on the landscape, scroll down and click Next. You will be taken to the Classified Outputs page which displays the landscape area, the outputs from the initial model run, and now the classified outputs (outlined in red in the image below) based on the percentages we have specified.

When you have completed the run you may scroll down and click Finish.

Run Summary

Upon clicking Finish you will be taken to the Run Summary page. Descriptive information on the run, and area of interest, are displayed at the top of the summary. At the bottom of the page is the Data Sets section.

From the Data Sets section you may choose Save As to save each data set in the IFTDSS Saved Data Sets section. You may also elect to Download the data as a raster, shapefile, or LCP file, for use outside of IFTDSS.

When finished, you may scroll down and select Back to Project.

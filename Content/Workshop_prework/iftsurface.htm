Running IFT-Surface Fire Behavior

In this tutorial we'll produce fire behavior outputs using IFT-Surface in the Prescribed Burn Planning workflow, assuming a pre-treatment landscape, and typical fall environmental conditions. We'll also see how outputs from this run can be labeled so they may be imported into a burn plan document later if we elected.

Inputs to run IFT-Surface include:

  • Fire Behavior Fuel Model
  • 1,10, and 100 hour fuel moistures
  • live herbaceous and woody fuel moistures
  • midflame windspeed
  • slope
  • aspect
  • flanking fire direction
  • elapsed time

Outputs will include:

  • Spread rates for heading, backing, and flanking fire
  • Heat per unit area
  • Fireline intensity for heading, backing, and flanking fire
  • Flame length for heading, backing, and flanking fire
  • Reaction intensity
  • Spread direction for heading, backing, and flanking fire
  • Spread distance for heading, backing, and flanking fire
  • Residence time
  • Effective windspeed

Initiating the Run

From the IFTDSS Dashboard, go to the project summary page for the project we created in Tutorial 1, via the Projects tab, or the Manage My Projects link.

From the Project Summary page, scroll down and select Create New Run. From the list of tasks, select Prescribed Burn Planning>Fire Behavior> and Calculate surface fire behavior (IFT-Surface).

Once selected, give the run a descriptive name such as 'Squaw Crk. Rx Fire' and click Next.

Configuring the Run

When configuring IFT-Surface you may create multiple simulations or stands to describe different fuel models, or different environmental or fuel parameters. This can be especially useful if you plan on using the IFTDSS burn plan feature later on, because you may create different simulations for different prescription scenarios and import these into the plan. For example, if you wanted to evaluate 4 different windspeeds, each with 4 different fuel moisture scenarios, you could conduct a run with 16 simulations.

For this example, we'll create 4 simulations, and assign each one a different windspeed. Under Number of Stands or Simulations, enter '4' and click Next to continue.

Inputs

Next we'll populate the inputs for each simulation. We'll give each the same information with the exception of the windspeed parameter.

For each input enter:

  • Fire Behavior Fuel Model: TU5
  • 1 hr fuel moisture: 8%
  • 10 hr fuel moisture: 10%
  • 100 hr fuel moisture: 16%
  • Live herbaceous fuel moisture: 45%
  • Live woody fuel moisture: 100%
  • Wind direction: 200 degrees
  • Slope: 20%
  • Aspect: 180 degrees
  • Flanking Fire Direction: 90 degrees
  • Elapsed Time: 2 hours
  • For midflame windspeed use: 6,8,9 and 10 mph, respectively.
   Note   

In this case we elected to use TU5 as the fire behavior fuel model, because it was the most common on our landscape. To find this information, hold down the Control key on your keyboard and click the Data tab at the top of IFTDSS to open Saved Data Sets in a new window. From the new window, find our data set and select edit under Actions. When the data set opens, click the Landscape Statistics key in the Mapping toolbar, make sure the 'Parameter' is set to 'fuel model', and 'Select Polygon' is set to 'Whole dataset', and click Go. This will generate a list of the fuel models on the landscape, and the acreage and percent cover associated with each one.

When completed your inputs should resemble the image below. Scroll down and click Next.

Outputs

Before looking at the outputs, note that you can toggle between tablular view, and graphical view using the Views options to the left of the results.

 

Additionally, if you want to export these outputs as a CSV spreadsheet to use outside of IFTDSS, that can be done by scrolling down to the bottom of the page and clicking Export Table (CSV).

Another feature of IFT-Surface is the ability to label output results so they may be imported into a burn plan later. This is done via the Mark simulations for use with Burn Plan section. For example, we set this run up to perform 4 simulations, each looking at a different windspeed (6,8,9 and 10 mph). If these were the range of windspeeds for a prescribed burn plan we were writing, we could label each output accordingly.

Use the dropdown menu to label each simulation as low, optimal, high, and maximum fire behavior accordingly. When finished, that section should resemble the image below. Remember order in which windspeed inputs were entered: windspeeds 6,8,9,10, correspond to low, optimal, high, and maximum fire behaviors, respectively.

You may also notice that there are three different prescription options in the prescription dropdown (prescription 1,2,or 3). To save time in this tutorial, we'll assume that each of the windspeeds entered is part of a single prescription. However, when using IFTDSS for your own future projects, it may be useful to know that you may label these outputs as different prescriptions. This allows you to create simulations and label outputs for up to three different prescription scenarios, which could then be input into the prescribed burn plan template. The prescribed burn plan template will be introduced during the 'Test Your Knowledge' portion of this online course.

   Note   

In IFTDSS, the Prescribed Burn Plan template and associated fire behavior modules allow the user to create and populate up to 3 different prescriptions per one burn plan

Scrolling down, you will see the outputs for each simulation displayed in a table.

Take a few minutes to look through the results. When finished evaluating the outputs, scroll down and click Finish to proceed to the Run Summary.

Run Summary

From the run summary page you can download a CSV file of the outputs, as well as a chart representing each output.

When finished with the run you may scroll down and select Back to Project.
